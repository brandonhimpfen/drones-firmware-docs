# Glossary

**Actuator allocation** — Conversion of control demands into motor or servo outputs.

**Arming** — Transition that allows motor or actuator outputs to become active.

**Bootloader** — Minimal firmware used to validate, install, or start the main firmware image.

**Control loop** — Repeated computation that compares desired and measured state and produces corrective output.

**Failsafe** — Predefined behavior triggered by abnormal or unsafe conditions.

**Firmware manifest** — Machine-readable metadata describing a firmware artifact.

**Geofence** — Virtual operational boundary.

**HIL** — Hardware-in-the-loop testing using real hardware with simulated environment signals.

**SITL** — Software-in-the-loop testing using simulated vehicle and environment models.

**Watchdog** — Hardware or software supervisor that detects stalled execution and triggers recovery.
