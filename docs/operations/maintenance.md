# Firmware Maintenance

Firmware maintenance includes updates, parameter audits, log review, calibration review, and compatibility checks.

## Maintenance schedule examples

| Interval | Firmware task |
|---|---|
| Before every flight | Version, parameter, health, and log checks |
| After every flight | Log extraction and anomaly review |
| Weekly | Parameter drift review and update audit |
| Monthly | Firmware compatibility and known issue review |
| Per release | Regression validation and rollback verification |

## Maintenance records

Keep records of firmware versions, parameter hashes, update dates, operators, incidents, and test evidence.
