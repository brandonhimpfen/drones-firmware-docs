# Firmware-Aware Pre-Flight Procedure

Pre-flight procedures should include firmware-specific checks.

## Firmware checks

- Confirm firmware version
- Confirm parameter profile
- Confirm hardware target
- Confirm sensor health
- Confirm calibration status
- Confirm geofence status
- Confirm logging enabled
- Confirm failsafe settings
- Confirm battery thresholds
- Confirm mission upload, if applicable

## Abort conditions

Abort flight if firmware version, parameter profile, sensor health, arming checks, battery status, or failsafe configuration do not match the approved configuration for the operation.
