# Subsystem Documentation Template

## Purpose

Describe what the subsystem does and why it exists.

## Scope

State what is included and excluded.

## Inputs

List data, parameters, sensor values, commands, or events consumed.

## Outputs

List messages, actuator commands, logs, status values, or state changes produced.

## Timing

Document rate, deadline, jitter expectations, priority, and failure impact.

## Parameters

List related parameters or point to the parameter catalog.

## Failure behavior

Describe detection, response, logging, and recovery.

## Test coverage

Link to relevant unit, integration, simulation, bench, HIL, or flight tests.

## Version history

Track behavior changes by firmware version.
