# Contributing

Contributions should improve documentation clarity, correctness, safety, or maintainability.

## Contribution types

- New subsystem documentation
- Better templates
- Test checklist improvements
- Schema improvements
- Python tooling improvements
- Corrections and clarifications

## Pull request expectations

- Keep documentation practical and version-aware.
- Avoid unsafe or harmful operational guidance.
- Include examples where useful.
- Update indexes when adding new pages.
- Keep Python examples dependency-light unless necessary.
